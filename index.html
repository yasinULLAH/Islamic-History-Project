About The Project

The Islamic History Portal is a single-file PHP web application designed to provide users with a rich, interactive platform to explore historical events (both Islamic and general), search and read Quranic ayahs with Urdu translation, and browse a collection of Hadiths. The application features role-based access, content management, multi-language support (English/Urdu), and gamification elements like points and badges.

This project is built with PHP and SQLite, utilizing Bootstrap 5 for responsive design, Leaflet.js for interactive maps, and Chart.js for visualizations. It aims to be an accessible and educational resource for anyone interested in Islamic history and teachings.

Key Features

Entire backend and frontend logic contained within one PHP file for ease of deployment.

SQLite Database

Admin, Ulama, User, and Public roles with distinct permissions.

Interactive event maps (Leaflet.js), location suggestions, "Zoom to My Location" and live tracking.

Content Management

Categorized events, infographic-style timeline, user submissions with approval workflow.

Religious Content Hub

Searchable Quran and Hadith databases, with content linking to historical events.

Multi-Language & RTL

English/Urdu interfaces with full RTL support for Urdu.

Gamification

Points system and achievement badges to encourage user engagement.

Modern UI/UX

Responsive design (Bootstrap 5) with Light/Dark mode themes.

Admin Utilities

User management, full content control, badge management, and database backup/restore.

Secure & Robust

Setup Instructions

Setting up the Islamic History Portal is designed to be straightforward:

  1. Web Server & PHP: A web server (Apache, Nginx) with PHP (7.4+ recommended) and the pdo_sqlite extension enabled.
  2. Download Files: Get index.php (the application) and data.AM (Quran data).
  3. File Placement: Place both index.php and data.AM in the same web-accessible directory.
  4. Directory Permissions: The web server user must have **write permissions** for the directory containing index.php. This allows the application to create the islamic_history_app.sqlite database and error.log file.
  5. Access: Navigate to index.php in your browser.

First Run - Quran Data Import: The application will automatically parse data.AM and populate the Quran ayahs on its first run if the database is empty. The data.AM file format is:
[Arabic Ayah] ترجمہ: [Urdu Translation]
س [3-digit Surah] آ [3-digit Ayah] (each entry on a new line).

How to Use The Portal

Public Users (Read-Only Access)

Visitors can freely explore approved content: browse historical events (Islamic/General), view events on the interactive map and timeline, search Quranic ayahs (Arabic/Urdu), and read Hadiths. Language and theme (Light/Dark) can be switched.

Registered Users

Upon registration, users gain the ability to contribute by suggesting new historical events and Hadiths (which await moderation). They can also bookmark content for easy access and earn points/badges for their contributions, viewable on their profile.

Ulama (Content Moderators)

Ulama have enhanced permissions to moderate user-submitted content (approve/reject events and Hadiths) via the Admin Panel. They can also add religious content directly (often auto-approved) and link ayahs/Hadiths to events.

Admin (Full System Control)

Administrators possess complete control, including all Ulama capabilities, plus: user management (roles, deletion), full content editing/deletion, badge system management, and critical database backup and restore functionalities.
